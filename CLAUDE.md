# CLAUDE.md - D-Booth AI 开发规范

> 本文档定义 D-Booth 项目的 AI 辅助开发规范、工作流程、代码质量标准与最佳实践。  
> 所有使用 AI 工具的开发者必须遵循此规范。

**版本**: 2.0.0 | **更新**: 2026-07-02 | **范围**: Backend/Frontend/Runtime

---

## 核心原则

1. **正确性优先** - 功能符合需求，边界情况完整处理
2. **可维护为本** - 代码清晰，架构合理，文档完善
3. **效率至上** - 最小化返工，批量操作，并行执行
4. **安全第一** - 数据隔离，输入验证，权限控制
5. **性能兼顾** - 响应迅速，资源高效，可扩展

---

## 标准工作流

对于所有非平凡任务，严格遵循六步法：

```
理解 → 规划 → 探索 → 实现 → 验证 → 完成
```

### 1. 理解 (Understand)

**目标**: 深入理解需求和上下文

- ✅ 明确用户实际目标（而非表面需求）
- ✅ 识别缺失信息并主动澄清
- ✅ 评估任务范围与复杂度
- ❌ 基于假设直接开始编码
- ❌ 跳过需求澄清直接提供方案

### 2. 规划 (Plan)

**目标**: 制定全局实现方案

- 确定需要修改的所有模块
- 识别上游调用者和下游依赖
- 评估对现有功能的影响
- 规划文件修改的顺序和批次

**输出物**: 实施计划、受影响文件列表、关键决策点

### 3. 探索 (Explore)

**目标**: 充分理解现有代码库

**搜索策略**:
```
Glob (定位候选文件) → Grep (精确匹配内容) → Read (深入理解实现)
```

**必须理解**:
- 项目结构与模块关系
- 依赖图与调用链
- 公共 API 与类型定义
- 现有实现模式与惯例

**禁止事项**:
- ❌ 猜测文件位置（必须搜索确认）
- ❌ 假设 API 签名（必须读取源码）
- ❌ 基于记忆编码（代码可能已变更）

### 4. 实现 (Implement)

**目标**: 高质量批量完成代码变更

**实施原则**:
1. **批量编辑** - 收集所有变更点后统一修改
2. **完整性** - 单次实现覆盖所有受影响文件
3. **一致性** - 保持项目现有风格和模式
4. **原子性** - 每个提交是独立可用的功能单元

**禁止模式**:
```
❌ 错误: Read → Edit → Read → Edit → Read → Edit
✅ 正确: Read + Read + Read → Edit + Edit + Edit
```

### 5. 验证 (Verify)

**目标**: 确保变更正确且无破坏

**必做检查**:
```bash
# 1. 编译/类型检查
mypy app/                    # Python
pnpm typecheck               # TypeScript
dotnet build                 # C#

# 2. 测试
pytest tests/ -v             # Python
pnpm test                    # TypeScript
dotnet test                  # C#

# 3. 代码质量
black . && isort .           # Python
pnpm lint                    # TypeScript
dotnet format                # C#
```

**验证失败处理**:
- 立即修复错误（不留待后续）
- 分析失败根因（不仅修复表象）
- 更新测试以防回归

### 6. 完成 (Complete)

**交付清单**:
- [ ] 所有代码已提交
- [ ] 测试全部通过
- [ ] 文档已同步更新
- [ ] CHANGELOG 已更新
- [ ] 版本号已递增（根据 SemVer）
- [ ] Git 提交消息符合规范
- [ ] 清理临时文件和调试代码

## 工具使用策略

### 工具选择决策树

```
查找文件?        → Glob (pattern matching)
搜索内容?        → Grep (content search)
理解实现?        → Read (file reading)
修改代码?        → Edit (precise replacement)
创建文件?        → Write (file creation)
执行构建/测试?   → Bash (command execution)
```

### 批量操作原则

**✅ 优先使用**:
- 并行 Glob + Grep + Read（信息收集）
- 批量 Edit（多文件修改）
- 单次 Bash（复合命令）

**❌ 避免使用**:
- 逐个读取文件
- 频繁切换工具
- 重复调用相同工具

### 并行工具调用示例

```xml
<!-- ✅ 正确：并行执行独立任务 -->
<function_calls>
  <invoke name="Read">
    <parameter name="file_path">D:/path/file1.py
