# Changelog

所有对本项目的重要更改都将记录在此文件中。

格式基于 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.0.0/)，
版本遵循 [语义化版本](https://semver.org/lang/zh-CN/)。

## [Unreleased]

### 变更
- 前端本地工作流统一使用 `npm ci`、`npm run dev`、`npm run typecheck` 和 `npm run build`，与 CI 和 `package-lock.json` 保持一致。
- 移除前端 pnpm workspace 配置，并让仓库卫生检查阻止 pnpm/yarn 锁文件回归。
- 移除冗余的前端 `VERSION` 文件，并让仓库卫生检查阻止额外版本源回归。
- 移除冗余的 Runtime `VERSION` 文件，并让仓库卫生检查阻止 Runtime 额外版本源回归。
- 模板编辑器新增保存到后端模板库能力，复用当前活动团队或用户默认团队，并保留 JSON 导出作为独立操作。
- 模板中心新增后端已保存模板列表入口，可打开团队模板继续编辑并更新。
- 模板创建流程参考离线版基础模板新增快速打印版式，可从模板中心或空画布一键生成 1/2/3/4 张、单版/双条、横竖版照片框布局。
- 模板中心支持将已保存模板设为当前拍照/打印模板，模板编辑器保存后也会自动设为当前打印模板。
- 相机页显示当前已选模板，打印预览页按当前模板渲染照片框、文本、形状、图片占位和二维码占位。
- DSLR 拍摄在真实活动上下文中会直接创建后端照片记录，并进入当前拍摄流，后续模板预览、打印和分享可复用同一个 `photo_id`。
- 打印预览页进入模板库选择模板后会返回打印预览，便于顾客直接确认同一张出纸预览已更新。
- 模板中心已保存模板新增复制和删除入口，降低自定义模板复用和清理成本。
- 模板编辑器支持上传底图和装饰图片，模板中心缩略图可显示已保存底图，降低从离线版/设计稿复刻自定义模板的操作成本。
- 模板编辑器空画布新增“上传底图 / 添加照片框 / 快速版式”引导，并强化照片框视觉提示，降低顾客从设计稿开始制作模板的操作成本。
- 模板编辑器新增底图锁定、照片框图层编号和照片号快速切换，降低误改底图和多照片框配置成本。
- 模板编辑器新增画布边界约束和中心/边缘吸附参考线，拖动或缩放照片框时更容易对齐版式并避免元素被拖出画布。
- 模板编辑器画布区域支持鼠标滚轮按预设档位缩放，并固定左右侧栏宽度，减少编辑自定义模板时的缩放菜单操作和布局挤压。
- 模板编辑器底图现在作为可排序图层参与上下移动，便于把装饰图片放到底图前后。
- 模板编辑器拖动和缩放元素时先使用临时交互布局，松手后再写入撤销记录，避免一次拖动产生大量无效 undo 步骤。
- 模板编辑器保存成功后返回模板中心并高亮刚保存的模板，方便顾客继续选择、复制或进入拍摄使用。
- 模板中心推荐/热门静态卡片改为创建对应快速版式，不再把顾客带到空白模板编辑器。
- 从相机或打印预览进入模板中心时，快速/推荐/热门版式可直接设为当前打印模板并返回原页面，避免顾客被迫进入编辑器。
- 从相机或打印预览进入模板中心选择快速版式时，已登录且能确定团队的流程会先保存为后端模板，再带着真实模板 UUID 返回原页面，确保预览和真实打印使用同一模板身份。
- 相机拍照后的打印动作改为进入打印预览，并提供选择/更换模板入口；打印预览未选模板时会明确引导先去模板库生成成品预览，模板中心在选择模式下显示返回目标和当前模板。
- 相机页和打印预览页现在会显示当前模板所需照片数与已拍数量；未选模板会先进入模板选择，照片不足会引导继续拍摄，补齐后再进入最终打印预览。
- 相机页拍照后的美颜、选模板、打印预览操作区改为常驻，避免顾客拍完后错过下一步；多照片模板会显示每个照片槽位的完成状态。
- 相机取景画面会按当前模板叠加照片框位置和照片编号，让顾客在拍摄时就能看到每张照片会落到模板哪里。
- 打印预览页会在预览下方和打印设置中展示同一套模板照片槽位状态，缺照片时可直接返回相机补拍。
- 打印任务支持携带当前模板，真实打印执行会按前端模板 layout 合成出纸图，并优先使用同一拍摄 session 的照片填充多照片框。
- 打印预览页没有本次拍摄照片时不再展示演示图或允许提交打印，改为引导返回拍照。
- 打印预览页移除未上传照片的本地模拟打印分支；照片必须上传到后端并具备真实 `photo_id` 后才允许提交打印任务。
- 打印预览页移除静态打印历史和固定今日打印数量，底部摘要改为展示真实系统打印队列。
- 打印预览页移除未接入真实打印任务的纸张选择、色彩优化和内联校准控件，出纸尺寸改为按当前模板只读展示。
- 后端打印执行在无可用打印机或打印机不可用时会将任务标记为失败，不再模拟等待后写入完成状态。
- 后台照片处理和拼图任务在未配置 R2 时会真实写入本地 `/uploads` 文件，不再返回不存在的处理结果 URL。
- 运营控制台将未实现的打印校准占位入口改为真实可用的打印中心入口，避免现场开机检查进入死路。
- 活动管理页“新建活动”接入后端创建接口，并让状态筛选按钮真实过滤活动列表。
- 活动列表移除只弹提示的活动设置按钮，系统设置页将未接入的主题、会员、积分和账号安全操作改为只读状态展示。
- 系统设置页移除未接入账号与订阅能力的假会员到期、假 AI 积分和假最近登录文案，改为真实只读状态说明。
- 系统设置页“关于”信息改为展示前端真实版本号，并移除伪造设备 ID、商业授权和构建号。
- 锁屏不再内置 `1234` 默认 PIN；未配置 PIN 或仍使用默认演示 PIN 时会阻止解锁并提示先配置非默认 PIN。
- 系统设置页新增锁屏 PIN 配置入口，支持设置 4-6 位非默认数字 PIN，已有 PIN 时修改需输入当前 PIN。
- 活动管理页移除后端无数据时的演示活动回退、静态汇总和假最近活动/收入图表；未登录或无活动时展示真实空状态，演示活动不能再进入拍照。
- 工作台“开始拍摄”改为进入活动管理选择真实活动，并移除硬编码设备状态、趋势图和运营统计。
- 数据统计页移除默认演示业绩、假趋势和假活动分布；未登录、无团队或接口无数据时展示真实空状态，趋势图等待后端趋势接口后再开放。
- GoPro 页面移除本地演示设备和定时模拟连接，扫描、连接、状态读取和断开连接改为调用后端真实 `/gopro` 接口。
- 相册页移除“开发中”分享入口，将单张/批量下载改为真实浏览器下载，并让分类筛选真实过滤照片列表。
- 分享页移除本地假分享链接、演示邮件/短信、AirDrop/微信/云端相册假成功入口和静态假统计；未上传后端照片时会明确阻止分享。
- 美颜页在进入打印或分享前会将已处理的美颜结果写回当前拍照流程，确保后续使用的是顾客看到的处理后照片。
- 美颜页贴纸现在会正确显示 emoji 贴纸，并在进入打印或分享前合成为真实输出图片。
- 美颜页隐藏尚未形成真实输出闭环的滤镜、调色、文字和裁剪入口，避免顾客进入不可完成的操作路径。
- AI 功能中心只展示已接入的美颜和背景替换入口，并移除未接入功能卡片和充值假按钮。
- 清理前端常量文件中已无引用的演示活动、假统计、假分享方式、假打印机和打印预览占位数据，降低后续误用风险。

- 修复多照片模板在前端预览和后端渲染缺少第 2/3/4 张照片时复用第一张照片的问题；后端打印任务会拒绝 session 照片数量不足的模板打印。
- 修复相机页“重拍”只隐藏拍后操作但不移除刚拍照片的问题，避免坏照片继续占用模板照片槽位。

### 优化
- Booth 注册和心跳时间戳改为 timezone-aware UTC 时间，避免 Python 3.12 `datetime.utcnow()` 弃用警告。
- Redis 缓存连接失败后短路降级，避免本地开发和测试在 Redis 不可用时反复阻塞。

## 版本说明

### 版本号格式：MAJOR.MINOR.PATCH

- **MAJOR**: 破坏性变更，不向后兼容
- **MINOR**: 新功能，向后兼容
- **PATCH**: Bug 修复，向后兼容

### 变更类型

- `新增` - 新功能
- `变更` - 现有功能的变更
- `弃用` - 即将移除的功能
- `移除` - 已移除的功能
- `修复` - Bug 修复
- `安全` - 安全漏洞修复
- `性能` - 性能改进
- `文档` - 文档更新
