## [Unreleased]

### 安全
- 公开分享码端点在读取路径上强制校验过期时间：过期分享返回 404，不再依赖小时级 Celery 清理任务兜底；未部署 Celery beat 时过期链接也不会继续暴露访客照片。
- 取消订阅改为故障封闭：Stripe 取消调用失败时返回 HTTP 502 且本地订阅状态保持不变，不再出现本地显示已取消而 Stripe 继续扣款的静默分歧；Stripe 同步调用移入线程池避免阻塞事件循环。
- Share 的 photo 反向关系改为 `noload`，公开分享查询不再级联加载整个活动的照片/会话/签名数据，并纳入叶子关系配置回归守护。
- Runtime 会话启动改为原子插入：相同 sessionId 仅在 event、mode、device 身份完全一致时幂等返回既有状态，不同身份返回 HTTP 409 / `SES_CONFLICT`，不再覆盖或重置已有会话。
- Runtime Shot ID 改为全局不可重用；仓储拒绝跨会话重归属，捕获使用唯一临时 JPEG 和不可覆盖移动，数据库或文件冲突返回 HTTP 409 / `SHT_CONFLICT` 并保留原归属与文件。
- Runtime 拍照文件路径现在验证会话和 Shot 标识并强制落在配置的 captures 根目录内，阻止路径穿越覆盖任意文件。
- 后端 JWT 依赖从 `python-jose` 迁移到 `PyJWT[crypto]`，并升级 MediaPipe 依赖以消除 `ecdsa` / `protobuf` 安全审计阻断。
- 禁用仓库内 Claude Stop hook 的自动版本递增、提交和 push 行为，并让仓库卫生检查阻止未验证自动提交配置回归。
- 外部触发器禁用本地程序执行动作，HTTP 回调目标会拒绝 localhost、私有网段、链路本地、保留地址和不可解析主机，避免触发器被配置成 RCE 或 SSRF 通道。
- 外部触发器批量更新会先完整校验新配置再替换旧配置，避免无效配置导致既有触发器被删除。

### 变更
- 新增账号登录/注册页并接入全局入口：顶栏与侧栏头像可点击进入登录（已登录时展示账户信息与登出）；此前前端不存在任何登录界面，本地或生产部署都无法通过 UI 进入登录态，所有需认证的运营功能不可用。
- 前端登出现在会调用后端 `/auth/logout` 撤销 refresh token（撤销失败时提示但仍清除本地令牌），不再只清 localStorage 留下有效的 refresh token。
- Booth 健康轮询单例化：新增 `BoothHealthProvider` 在应用根部维护唯一 8 秒健康轮询，顶栏与打印/运营页共享同一份数据；此前顶栏常驻实例与页面实例叠加会产生 2-3 路重复轮询。顶栏不再触发 5 秒打印队列轮询（队列由真正展示它的打印/运营页按需拉取）。
- 相册页接入分页加载：每页 60 张、支持"加载更多"，大活动照片不再被后端单次上限静默截断；页头明确显示已加载数量与是否还有更多。
- 统一 AI 开发规范为 `CLAUDE.md` 单一权威：删除与之冲突的 `PROTOCOL.md`（可用理念并入 CLAUDE.md，移除"永不收敛/无需用户确认"的自主执行协议），修复 CLAUDE.md 末尾截断的示例，新增真实质量门禁、版本同步清单与提交纪律章节。
- 路线图单源化：README 与 ARCHITECTURE 不再维护独立路线图（改为链接 ROADMAP.md）；ROADMAP 当前版本基线纠正为根 `VERSION` 权威，2027+ 版本改为未排期方向性规划，移除 NFT/区块链/元宇宙等未立项条目与虚构的季度承诺。
- 文档对齐真实工具链：移除不存在的脚本/命令引用（`seed_db.py`、`deploy.sh`、`scripts.create_admin_user`、`npm test`、`npm run generate-types`），明确前端当前无单元测试框架（门禁为 typecheck+build）、全量 mypy 非 CI 门禁；仓库卫生检查新增对应防回归规则。
- API.md 限流文档与实际中间件对齐（基于 IP 的分钟/小时双窗口、真实响应头、豁免路径）；SECURITY.md 移除虚构的 CVE 记录、未发生的第三方审计结论与占位 PGP/致谢内容。
- 删除前端从未填写的 Figma 模板占位文件 `guidelines/Guidelines.md`。
- 补充缺失的 `SMTP_USE_TLS` 配置项；分享测试邮件（`send_photo_email`）此前读取不存在的设置抛 AttributeError 并被吞成失败，SMTP 配置完整时也从未发送成功。
- 打印任务执行改为原子领取（compare-and-swap）：只有 PENDING/QUEUED 状态的任务能被领取进入 PRINTING，Celery 重试与后台执行并发时同一任务不再可能被重复打印或重复计数。
- 打印发送改用可反馈退出码的 PowerShell `Start-Process -Verb Print -Wait` 作为首选路径；`ShellExecute` 因异步无结果反馈且 `/d:` 参数常被忽略（可能打到默认打印机）降级为最后退路，打印失败不再被无条件标记为完成。
- R2 存储对象 URL 改用可配置的公开访问基址（`R2_PUBLIC_URL`，指向 pub-*.r2.dev 或自定义域）；S3 API 端点不支持匿名 GET，此前入库的照片 URL 在生产启用 R2 后全部不可访问、打印拉图必失败。上传、预签名和 Celery 异步上传统一走公开基址，未配置时回退端点形式（仅测试场景）。
- 实现 `R2StorageService.download_file`；道具合成（`POST /props/apply`）此前调用不存在的方法必抛 AttributeError/500，现可真实下载道具图并合成。
- 邮件发送改为故障封闭：`EmailService.send_email` 对已配置 SMTP 的发送失败抛异常（而非吞掉返回 False），四个邮件任务（欢迎/密码重置/活动邀请/照片分享）据此触发 Celery 重试；未配置 SMTP 时返回 skipped 状态而非谎报 sent。
- 移除打印机策略硬编码的开发机型号优选；常用打印机现在来自用户在打印页的真实选择并持久化，默认排序退回真实系统默认打印机。
- 修复打印页 30 秒硬超时把仍在打印的任务谎报为失败的问题；兜底延长到 3 分钟，超时只停止自动跟踪并引导查看真实打印队列，任务完成后兜底计时会被清理，页面卸载也会停止轮询。
- 移除相机预览失败时的婚礼样张回退和美颜页无照片时的模特头像回退；相机显示诚实的不可用占位，美颜页无照片时展示空状态并引导返回拍摄。
- 系统设置页水印与打印锐化标记为未接入并只读展示，不再让用户误以为配置会影响实际照片与打印输出。
- 顶栏真实数据化：活动名称显示当前所选活动并可点击进入活动管理，相机/打印机指示灯与名称来自真实 Booth 健康状态轮询，移除硬编码的假活动、假设备双绿灯、假 Pro 会员徽章和假用户头像；侧栏头像同步显示真实登录用户。
- 相册页从硬编码素材样张改为展示当前活动的真实后端照片：支持最新/最早排序、刷新、单张与批量真实下载；未登录、未选活动、无照片和加载失败均展示诚实状态。
- AI 功能中心移除硬编码的假使用统计、假热门榜、假积分余额和不实的 GPT-4V 徽章，改为明确的未接入说明。
- 前端照片上传迁移到统一 API 客户端：401 时自动刷新令牌并重试（根治拍摄上下文快照令牌过期后上传持续失败）、附带 CSRF 头、子路径部署下正确解析上传地址；上传保持不自动重试避免重复照片。
- 修复相机页视频模式录出黑屏视频的问题；录制绘制循环此前读取过期状态从未启动，现使用 ref 标记真实录制状态。
- 移除相机页无真实链路的美颜强度假滑杆和绿幕假开关；绿幕区块保留进入真实绿幕设置的入口，拍后美颜继续走美颜页真实处理。
- 修复 Runtime 重复取消会话返回 HTTP 500 的问题；重复取消现在幂等返回既有取消状态，已完成/失败会话的取消请求返回 HTTP 409 / `SES_INVALID_STATE` 且不改写终态。
- 修复 Runtime 同一会话连拍第二张被会话状态机拒绝并返回 HTTP 500 的问题；Countdown/Capturing 会话现在可按序捕获多张 Shot，终态会话拍摄返回 HTTP 409 / `SES_INVALID_STATE` 且不触碰相机或产生文件。
- Runtime 拍照不再把 JSON 文本伪装成 JPEG；只有已连接的 `ICameraPlugin` 写出指定路径且通过 JPEG 签名校验后才保存 Shot，未配置或伪成功统一返回 `CAM_DEVICE_NOT_READY` 且不改变会话状态。
- Runtime 打印和分享执行器在未配置真实适配器时改为故障封闭，作业持久化为失败并记录稳定错误码，不再通过写 JSON 文件伪装执行成功或创建假输出资产。
- 移除浏览器 Webcam 的模拟 ISO、快门、白平衡等参数读写；不支持的设置现在明确返回 501，前端禁用不可写控件且不再显示模拟读值，gphoto2 在读值未接入时仍可执行真实参数写入。
- 修复 DSLR 捕获错误路径覆盖 FastAPI 状态码模块的问题，空图像、无效活动和捕获异常现在返回受控错误响应。
- 清理后端 11 份无引用的阶段总结、优化报告和开发进度快照，继续以根目录 `PROJECT_STATUS.md` 与 `CHANGELOG.md` 作为当前项目事实来源。

- 后端 CI 失败诊断改为从 pytest JUnit XML 提取具体失败测试、异常消息和 traceback，避免 coverage 输出截断真正的失败摘要。
- 后端 CI 测试步骤在 pytest 失败时会把失败摘要写入 GitHub annotation，避免日志权限受限时只能看到 exit code。
- 模板中心推荐/热门版式改为使用真实 `TemplateLayout` 缩略图渲染，并让搜索、分类、比例、风格、场景和颜色筛选同时覆盖官方版式目录与已保存模板。

- 移除冗余的前端 `VERSION` 文件，并让仓库卫生检查阻止额外版本源回归。
- 移除冗余的 Runtime `VERSION` 文件，并让仓库卫生检查阻止 Runtime 额外版本源回归。

- 模板编辑器新增画布边界约束和中心/边缘吸附参考线，拖动或缩放照片框时更容易对齐版式并避免元素被拖出画布。
- 模板编辑器画布区域支持鼠标滚轮按预设档位缩放，并固定左右侧栏宽度，减少编辑自定义模板时的缩放菜单操作和布局挤压。
- 模板编辑器画布滚轮缩放改为直接绑定画布视口，并让画布保持固定尺寸不被容器压缩，减少缩放后版式观察偏差。
- 模板编辑器底图现在作为可排序图层参与上下移动，便于把装饰图片放到底图前后。
- 模板编辑器拖动和缩放元素时先使用临时交互布局，松手后再写入撤销记录，避免一次拖动产生大量无效 undo 步骤。
- 模板编辑器保存成功后返回模板中心并高亮刚保存的模板，方便顾客继续选择、复制或进入拍摄使用。
- 模板中心和模板编辑器读取、校验、快速创建和保存模板时会复用当前拍摄上下文 token，避免从真实活动进入后仍被误判未登录。

- 从相机或打印预览进入模板中心时，快速/推荐/热门版式可直接设为当前打印模板并返回原页面，避免顾客被迫进入编辑器。

- 模板中心移除常用版式/快速创建窗口，快速版式入口保留在模板编辑器元素库，避免模板中心出现无装饰的版式卡片影响顾客判断成品效果。
- 模板中心已保存模板缩略图改为按真实模板 layout 渲染底图、照片槽、文字、图片和基础元素，不再用固定假占位块替代自定义模板效果。

## 版本说明

### 版本号格式：MAJOR.MINOR.PATCH

- **MAJOR**: 破坏性变更，不向后兼容
- **MINOR**: 新功能，向后兼容
- **PATCH**: Bug 修复，向后兼容

### 变更类型

- `新增` - 新功能
- `变更` - 现有功能的变更
- `弃用` - 即将移除的功能
- `移除` - 已移除的功能
- `修复` - Bug 修复
- `安全` - 安全漏洞修复
- `性能` - 性能改进
- `文档` - 文档更新
